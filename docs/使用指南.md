## 🚀 使用步骤

### 第一步：安装依赖

```bash
pip install litemapy Pillow pyvista numpy pandas jinja2 tqdm pyyaml
```

### 第二步：配置GitHub仓库

1. 访问 https://github.com/new
2. 创建新仓库 `litematic-manager`
3. 选择Public或Private
4. 不要初始化README、.gitignore或LICENSE

### 第三步：连接远程仓库

```bash
cd litematic-manager

# 添加远程仓库
git remote add origin https://github.com/yourusername/litematic-manager.git

# 推送代码
git push -u origin main
```

### 第四步：配置GitHub Pages

1. 进入仓库Settings
2. 找到Pages设置
3. 选择Source为`Deploy from a branch`
4. 选择Branch为`main`
5. 选择Folder为`docs/site`
6. 点击Save

### 第五步：上传litematic文件

```bash
# 复制litematic文件到对应分类
cp "你的文件.litematic" litematic_files/建筑/

# 提交并推送
git add litematic_files/
git commit -m "添加新文件: 你的文件.litematic"
git push origin main
```

### 第六步：本地渲染（可选）

如果你想本地渲染预览图：

```bash
# 渲染单个文件
python scripts/local_render.py --file "litematic_files/建筑/你的文件.litematic"

# 渲染整个分类
python scripts/local_render.py --category "建筑"

# 渲染所有文件
python scripts/local_render.py
```

### 第七步：生成网站（可选）

如果你想手动生成静态网站：

```bash
python scripts/generate_site.py
```

### 第八步：上传到GitHub（可选）

如果你想手动上传到GitHub：

```bash
python scripts/auto_upload.py --file "litematic_files/建筑/你的文件.litematic" --category "建筑"
```

### 第九步：访问网站

等待几分钟后，访问：
```
https://yourusername.github.io/litematic-manager/
```

## 🔧 插件集成

### 修改upload_command.py

在插件的 `upload_command.py` 中添加自动渲染和上传功能：

```python
async def _render_and_upload(self, file_path: Path, category: str):
    """渲染并上传到GitHub"""
    try:
        # 导入本地渲染脚本
        import sys
        scripts_path = Path(__file__).parent.parent / "scripts"
        sys.path.insert(0, str(scripts_path))
        
        from local_render import LocalRenderer
        from auto_upload import AutoUploader
        
        # 1. 本地渲染
        yield event.plain_result("正在渲染预览图...")
        renderer = LocalRenderer()
        result = renderer.render_file(file_path)
        
        if not result:
            yield event.plain_result("渲染失败")
            return
        
        # 2. 生成元数据
        yield event.plain_result("正在生成元数据...")
        self._update_metadata(result)
        
        # 3. 生成静态网站
        yield event.plain_result("正在生成网站...")
        from generate_site import SiteGenerator
        site_gen = SiteGenerator()
        site_gen.generate_all()
        
        # 4. 上传到GitHub
        yield event.plain_result("正在上传到GitHub...")
        uploader = AutoUploader(repo_path=".")
        uploader.commit_and_push(f"添加文件: {file_path.name}")
        
        yield event.plain_result("✓ 上传完成！")
        yield event.plain_result(f"访问地址: https://yourusername.github.io/litematic-manager/")
        
    except Exception as e:
        yield event.plain_result(f"✗ 处理失败: {e}")

def _update_metadata(self, result: Dict[str, Any]):
    """更新元数据"""
    import json
    from pathlib import Path
    
    metadata_file = Path("docs/data/metadata.json")
    metadata_file.parent.mkdir(parents=True, exist_ok=True)
    
    # 加载现有元数据
    if metadata_file.exists():
        with open(metadata_file, 'r', encoding='utf-8') as f:
            metadata = json.load(f)
    else:
        metadata = {
            "generated_at": datetime.now().isoformat(),
            "total_files": 0,
            "files": {},
            "categories": {}
        }
    
    # 更新文件元数据
    file_key = result["file_path"]
    metadata["files"][file_key] = result["metadata"]
    metadata["files"][file_key]["previews"] = result["previews"]
    metadata["files"][file_key]["materials"] = result["materials"]
    
    # 更新分类统计
    category = result["category"]
    if category not in metadata["categories"]:
        metadata["categories"][category] = {
            "name": category,
            "file_count": 0,
            "files": []
        }
    
    metadata["categories"][category]["file_count"] += 1
    metadata["categories"][category]["files"].append(result["metadata"])
    
    # 更新总数
    metadata["total_files"] = len(metadata["files"])
    metadata["generated_at"] = datetime.now().isoformat()
    
    # 保存元数据
    with open(metadata_file, 'w', encoding='utf-8') as f:
        json.dump(metadata, f, ensure_ascii=False, indent=2)
```

## 📊 工作流程

### 完整流程

```
用户通过Bot上传文件
    ↓
插件保存文件到本地
    ↓
触发本地渲染脚本
    ↓
调用插件渲染功能
    ↓
生成2D/3D预览图
    ↓
分析材料统计
    ↓
生成元数据
    ↓
生成静态网站
    ↓
自动上传脚本
    ↓
Git提交并推送
    ↓
GitHub Pages自动更新
    ↓
用户通过Web访问查看
```

### 简化流程（不使用插件联动）

```
1. 手动上传litematic文件到litematic_files目录
    ↓
2. 运行本地渲染脚本
    ↓
3. 运行网站生成脚本
    ↓
4. 运行自动上传脚本
    ↓
5. 访问GitHub Pages
```

## 🐛 常见问题

### 1. 渲染失败

**问题**：本地渲染失败

**解决方案**：
1. 检查Python依赖是否安装
2. 检查litematic文件是否损坏
3. 检查插件路径是否正确
4. 查看错误日志

### 2. 上传失败

**问题**：Git推送失败

**解决方案**：
1. 检查Git配置是否正确
2. 检查网络连接
3. 检查GitHub仓库权限
4. 查看Git错误日志

### 3. 网站无法访问

**问题**：GitHub Pages未部署

**解决方案**：
1. 检查GitHub Pages设置
2. 等待几分钟让部署完成
3. 检查文件路径是否正确
